# 🌱 Study Sprout

**Study Sprout** is a cute plant-themed task management web app built using **HTML, CSS, and JavaScript**.
It helps users add, complete, delete, and organize daily tasks in a visually engaging “garden” interface, where unfinished tasks appear as seedlings and completed tasks bloom into flowers.

This project was built as a **frontend practice project** to strengthen my understanding of **responsive UI design, advanced CSS layouts, and real-world JavaScript fundamentals**.

---

# ✨ Project Overview

Study Sprout is a **single-page to-do application** designed with a cozy pastel garden theme.
The goal of the project was not just to create a task manager, but to build a **complete responsive frontend mini-product** that demonstrates both **layout/styling skills** and **frontend logic implementation**.

Users can:

* add new tasks
* mark tasks as completed
* delete tasks
* filter tasks by **All / Pending / Completed**
* clear completed tasks
* track completion progress with a progress bar
* save tasks in **localStorage** so data remains after refresh
* view task creation dates
* receive toast notifications for actions

# 📚 Learning Focus Demonstrated in This Project

## 1) Advanced CSS Layouts

This project was used to apply **modern CSS layout techniques** rather than relying on basic block layouts.

### Concepts demonstrated:

* **Flexbox**

  * aligning header content
  * arranging task input section
  * structuring task cards and task actions
  * layout for filter controls and progress header

* **CSS Grid**

  * responsive stats section layout
  * clean card-based arrangement of information blocks

* **Positioning**

  * toast notification placement using fixed positioning
  * layout control for responsive UI sections

* **Structured component-based layout**

  * app card container
  * stats section
  * progress section
  * filter section
  * task display section
  * reusable task card design

This project helped me practice how to break a page into **modular UI sections** and style each one cleanly.

---

## 2) Responsive Design

Study Sprout was intentionally built as a **responsive project** so the layout adapts well across screen sizes.

### Concepts demonstrated:

* **media queries**

  * adjusting layout for tablets and mobile screens
  * stacking controls vertically on smaller widths
  * resizing buttons, headings, and task cards

* **mobile-friendly layout behavior**

  * input and button stack on smaller screens
  * stats cards move from multi-column to single-column layout
  * task cards shift into a vertical layout for better readability
  * toast adapts to smaller screens

* **responsive cards and sections**

  * stat cards
  * progress card
  * task cards
  * empty state section

* **responsive forms and controls**

  * task input form
  * filter buttons
  * clear completed button
  * action buttons inside task cards

This project demonstrates my understanding of designing an interface that is **usable, readable, and visually consistent across different device sizes**.
