# Pawfect Café — Responsive Cute Landing Page

A responsive single-page café website built using **HTML, CSS, and JavaScript** as part of my **Week 1 Frontend Development practice project**.
This project focuses on building a polished, mobile-friendly UI while applying core concepts of **modern HTML/CSS layout design and responsive frontend development**.

---

# Project Overview

**Pawfect Café** is a cute themed landing page for a fictional pet café where users can explore:

* a welcoming hero section
* an about section
* featured pet cards
* a café menu
* special event cards
* a reservation/contact form
* a responsive navigation bar
* a styled footer

The project is designed as a **frontend practice project** to strengthen layout building, section structuring, responsive design, and UI polish.

---

# Why I Built This Project

This project was created to practice and demonstrate my understanding of:

* advanced CSS layouts
* Flexbox and Grid
* positioning and sticky elements
* responsive design principles

* building responsive UI components such as navbars, cards, and forms

Instead of building a generic portfolio page, I chose a **cute themed café landing page** so I could focus on layout, aesthetics, responsiveness, and reusable frontend structure in a more creative way.

# Project Summary

**Pawfect Café** is a pastel-themed responsive landing page for a fictional pet café.
The website includes:

* **Hero section** with CTA buttons and visual content
* **About section** with highlight pills and informational cards
* **Featured Pets section** using responsive card layouts
* **Menu section** with food/drink cards
* **Events section** with event cards and CTA buttons
* **Contact / Reservation section** with a styled responsive form
* **Footer section** with quick links, contact details, and social links
* **Mobile navigation menu** with JavaScript toggle behavior

This project demonstrates how to structure and style a complete responsive landing page using only frontend technologies.

# Concepts Demonstrated in This Project

## 1) Advanced CSS Layouts

This project uses multiple section-based layouts to structure a complete landing page.
Each section was designed separately while keeping spacing, alignment, and visual consistency across the page.

### Demonstrated through:

* hero split layout
* about section content + card layout
* pets/menu/events card sections
* contact section two-column layout
* footer multi-column layout

---

## 2) Flexbox

Flexbox was used wherever horizontal/vertical alignment and flexible arrangement were needed.

### Used in:

* navbar layout
* hero button alignment
* hero feature cards wrapping
* contact mini-card content alignment
* footer content organization
* mobile navbar arrangement

### What this shows:

* alignment of items across rows
* spacing between content blocks
* wrapping elements on smaller screens
* building flexible UI sections without breaking layout

---

## 3) CSS Grid

Grid was used for structured multi-column sections, especially where repeated cards were required.

### Used in:

* hero main layout
* about section layout
* pets cards grid
* menu cards grid
* events cards grid
* contact form rows
* footer columns

### What this shows:

* creating responsive multi-column layouts
* organizing cards cleanly
* controlling column behavior across screen sizes
* switching from multi-column to stacked mobile layouts

---

## 4) Positioning

Positioning was used to enhance the visual design and improve layout control.

### Used in:

* floating info cards in the hero section
* category/age badges placed over cards
* menu/event labels
* dropdown mobile navigation positioning
* relative and absolute positioning inside cards

### What this shows:

* layering UI elements cleanly
* placing decorative/info components over images
* controlling placement without disturbing document flow

---

## 5) Sticky / Fixed Layouts

The navigation bar is designed as a **sticky header**, staying visible as the user scrolls through the page.

### Demonstrated through:

* sticky navbar at the top of the page
* scroll-friendly navigation experience

### What this shows:

* use of sticky positioning in real UI design
* improving navigation usability on longer pages

---

## 6) Responsive Design

The entire project is built to adapt across different screen sizes including desktop, tablet, and mobile.

### Responsive behaviors included:

* stacked hero layout on smaller screens
* responsive card grids
* mobile-friendly form layout
* footer columns collapsing on smaller screens
* navigation changing into a mobile menu

### What this shows:

* ability to build one design that works across devices
* layout restructuring instead of simple scaling
* attention to spacing, readability, and usability

---

## 7) Media Queries

Media queries were used extensively to control how the website behaves at different breakpoints.

### Used for:

* changing navbar behavior on mobile
* adjusting hero image sizes
* changing grid columns for cards
* stacking contact form fields
* resizing headings and spacing
* reorganizing footer layout

### What this shows:

* understanding of breakpoint-based layout design
* ability to optimize sections for different screen widths
* responsive control beyond default browser behavior

---

## 8) Mobile-First / Mobile-Friendly Thinking

Although the page was visually designed for larger screens first, the implementation strongly focuses on **mobile-friendly behavior** and responsive usability.

### Mobile-friendly considerations included:

* stacked layouts on small screens
* full-width buttons on mobile
* readable text sizing
* touch-friendly navbar toggle
* simplified mobile navigation
* single-column cards for narrow screens

### What this shows:

* awareness of mobile UX
* designing components that remain usable on smaller devices
* prioritizing readability and accessibility in responsive layouts

---

## 9) Responsive Navbar, Cards, and Forms

This project specifically demonstrates how to build responsive UI components that are commonly used in modern websites.

## Responsive Navbar

* desktop navigation with links + CTA

## Responsive Cards

## Responsive Form

* reservation form with grouped fields
* stacked layout on smaller screens
* consistent spacing and input styling

### What this shows:

* component-based thinking
* reusable card and form patterns
* practical frontend UI building skills
